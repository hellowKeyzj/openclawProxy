# 模型与能力图谱

本页介绍 AI 模型常见能力维度，帮助选择合适方案。

## 核心能力维度

| 维度 | 说明 | 常见场景 |
|---|---|---|
| 文本理解 | 阅读、总结、分类、抽取 | 报告处理、知识管理 |
| 代码能力 | 代码生成、调试、解释 | 开发助手、自动化脚本 |
| 多模态能力 | 图片/文档识别与理解 | OCR、图文问答 |
| 工具调用 | 调用外部 API 和系统工具 | 工作流自动化 |
| 长上下文 | 处理长文档与多轮对话 | 法务、审阅、研究 |

## 任务匹配建议

1. 日常问答与文案：通用对话模型
2. 结构化抽取：支持 JSON 输出与函数调用的模型
3. 代码场景：代码专项模型或强代码能力模型
4. 企业知识问答：模型 + 向量检索（RAG）

## 常用提示词模板

### 信息抽取

```text
你是信息抽取助手。请从以下文本中抽取：公司名、时间、金额。
输出 JSON，不要输出额外说明。
```

### 内容总结

```text
请将以下内容总结为 3 条要点，面向管理层，控制在 120 字以内。
```

### 任务分解

```text
请把目标拆成可执行步骤，每步包含：目的、输入、输出、验收标准。
```

## 质量评估要点

- 准确率：是否事实正确
- 稳定性：同类输入结果是否稳定
- 可解释性：结论是否给出依据
- 成本与时延：是否满足业务上线要求
